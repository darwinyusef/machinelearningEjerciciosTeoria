¿Qué es el Aprendizaje Automático?

El aprendizaje automático es la ciencia (y el arte) de programar computadoras para que puedan aprender de los datos. A diferencia de la programación tradicional, donde se escriben reglas explícitas, el ML permite a los sistemas descubrir patrones y mejorar con la experiencia. No todos los sistemas de aprendizaje automático son iguales. Se pueden clasificar según varios criterios clave que definen cómo aprenden y operan. Utiliza las pestañas para explorar estas clasificaciones y entender las diferencias fundamentales entre los distintos tipos de sistemas.

Aprendizaje por Lotes (Batch)

El sistema es incapaz de aprender de forma incremental; debe ser entrenado utilizando todos los datos disponibles. Esto se hace fuera de línea. Primero se entrena el modelo y luego se pone en producción, donde se ejecuta sin aprender más. Si se quieren añadir nuevos datos, hay que entrenar una nueva versión del modelo desde cero.

Aprendizaje en Línea (Online)

El sistema se entrena de forma incremental alimentándolo con instancias de datos secuencialmente, ya sea individualmente o en pequeños grupos llamados "mini-batches". Cada paso de aprendizaje es rápido y barato. Este método es ideal para sistemas que necesitan adaptarse rápidamente a cambios y para manejar grandes cantidades de datos.

Basado en Instancias

El sistema aprende los ejemplos de memoria y luego generaliza a nuevos casos comparándolos con los ejemplos aprendidos, utilizando una medida de similitud. Por ejemplo, para clasificar un nuevo correo, podría buscar los correos más similares en su base de datos y usar sus etiquetas para predecir la del nuevo.

Basado en Modelos

Se construye un modelo a partir de un conjunto de ejemplos de entrenamiento y luego se utiliza ese modelo para hacer predicciones. Este es el enfoque más común. Por ejemplo, un modelo de filtro de spam se entrena para "entender" qué características hacen que un correo sea spam, y luego aplica ese conocimiento para clasificar nuevos correos.

Definición de Aprendizaje Automático

El Aprendizaje Automático (ML) permite a las computadoras aprender sin ser programadas explícitamente. Se define como: "Un programa de computadora aprende de la experiencia $E$ con respecto a alguna tarea $T$ y alguna medida de rendimiento $P$, si su rendimiento en $T$, medido por $P$, mejora con la experiencia $E$."

Ejemplo del Filtro de Spam:

  • Tarea (T): Marcar correos electrónicos como spam.
  • Experiencia (E): El conjunto de datos de entrenamiento (ejemplos de spam y "ham" o no-spam).
  • Medida de Rendimiento (P): La proporción de correos electrónicos clasificados correctamente (precisión).

Ejemplo de Wikipedia (No ML): Si solo descargas todos los artículos de Wikipedia, tu computadora tiene más datos, pero no es "más inteligente" en ninguna tarea específica; esto no es aprendizaje automático.

Evolución y Aplicaciones del ML

El ML tiene una rica historia y se aplica en diversos campos:

Aplicaciones Tempranas:

  • Reconocimiento óptico de caracteres (OCR).
  • Filtro de spam (años 90).

Aplicaciones Modernas Cotidianas:

  • Indicaciones de voz.
  • Traducción automática.
  • Búsqueda de imágenes.
  • Recomendaciones de productos.

Chatbots de Última Generación:

  • ChatGPT, Gemini (anteriormente Bard), Claude, Perplexity, donde la IA es el servicio principal.

Ejemplos Específicos de Tareas y Técnicas (del PDF)

  • Clasificación de imágenes de productos: Clasificación de imágenes, usando Redes Neuronales Convolucionales (CNNs) o Transformers de visión.
  • Detección de tumores en escáneres cerebrales: Segmentación semántica de imágenes, usando CNNs o Transformers de visión.
  • Clasificación automática de artículos de noticias: Procesamiento del Lenguaje Natural (NLP), clasificación de texto, usando Redes Neuronales Recurrentes (RNNs), CNNs o Transformers.
  • Marcado automático de comentarios ofensivos: Clasificación de texto (NLP).
  • Resumen automático de documentos largos: Resumen de texto (NLP).
  • Estimación del riesgo genético de una enfermedad: Análisis de secuencias de ADN largas, usando Modelos de Espacio de Estados (SSMs).
  • Creación de un chatbot o asistente personal: Componentes de NLP, incluyendo comprensión del lenguaje natural (NLU) y módulos de preguntas y respuestas.
  • Previsión de ingresos de la empresa: Tarea de regresión (predicción de valores), usando modelos de regresión lineal, polinómica, SVM de regresión, Random Forest de regresión o Redes Neuronales Artificiales (ANNs). Para secuencias, RNNs, CNNs o Transformers.
  • Reacción de una aplicación a comandos de voz: Reconocimiento de voz, procesando muestras de audio con RNNs, CNNs o Transformers.
  • Detección de fraude con tarjetas de crédito: Detección de anomalías, usando Isolation Forests, Modelos de Mezcla Gaussiana o Autoencoders.
  • Segmentación de clientes por compras: Agrupamiento (clustering), usando k-means o DBSCAN.
  • Representación de conjuntos de datos de alta dimensión: Visualización de datos, usando técnicas de reducción de dimensionalidad.
  • Sistemas de recomendación de productos: Usando Redes Neuronales Artificiales entrenadas en secuencias de compras.
  • Construcción de un bot inteligente para un juego: Aprendizaje por Refuerzo (RL), entrenando agentes para maximizar recompensas (ej. AlphaGo).

Ventajas del ML sobre la Programación Tradicional

Mantenimiento del Filtro de Spam:

  • Enfoque Tradicional: Requiere una lista larga y compleja de reglas (ej. "4U", "crédito", "gratis"). Si los spammers cambian a "For U", el programador debe actualizar manualmente las reglas.
  • Enfoque de ML: El sistema aprende automáticamente qué palabras y frases son buenos predictores de spam. Si "For U" se vuelve frecuente en el spam marcado por los usuarios, el sistema lo detecta y lo marca sin intervención manual, adaptándose automáticamente a los cambios.

Problemas Demasiado Complejos:

El reconocimiento de voz es un ejemplo clave. Es inviable escribir reglas para miles de palabras, millones de personas, entornos ruidosos y docenas de idiomas. El ML permite que el algoritmo aprenda por sí mismo a partir de ejemplos.

Ayuda a los Humanos a Aprender (Minería de Datos):

Un filtro de spam entrenado puede revelar palabras o combinaciones de palabras que considera los mejores predictores de spam, descubriendo correlaciones inesperadas o nuevas tendencias, lo que lleva a una mejor comprensión del problema.

Clasificación de Sistemas de ML

Ejemplo de Filtro de Spam (Combinación de Criterios):

Un filtro de spam de última generación puede ser un sistema de aprendizaje supervisado (entrenado con ejemplos de spam y ham proporcionados por humanos), basado en modelos (utilizando una red neuronal profunda) y en línea (aprende sobre la marcha).

Aprendizaje Supervisado:

En el aprendizaje supervisado, el conjunto de entrenamiento incluye las soluciones deseadas, llamadas etiquetas (por ejemplo, un correo electrónico se etiqueta como "spam" o "ham").

Conceptos Fundamentales y Terminología

  • Conjunto de Entrenamiento: Los ejemplos que el sistema utiliza para aprender (ej. correos electrónicos marcados como spam/ham).
  • Instancia de Entrenamiento (o Muestra): Cada ejemplo individual en el conjunto de entrenamiento.
  • Modelo: La parte del sistema de ML que aprende y hace predicciones (ej. redes neuronales, random forests).
  • Etiquetas: Las soluciones deseadas en el aprendizaje supervisado (ej. "spam", "ham").
  • Precisión: Una medida de rendimiento común en tareas de clasificación, que es la proporción de elementos clasificados correctamente.

Flujo de Trabajo de un Proyecto de ML

El PDF ilustra un flujo de trabajo iterativo:

  1. Estudiar el problema.
  2. Entrenar el modelo de ML (usando datos).
  3. Evaluar el modelo.
  4. Analizar errores.
  5. Actualizar datos (si es necesario).
  6. Lanzar.
  7. Inspeccionar la solución (para obtener información).
  8. Repetir si es necesario (el ciclo de mejora continua).
